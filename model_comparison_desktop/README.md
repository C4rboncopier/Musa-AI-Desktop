# Musa AI Model Comparison Desktop

Standalone PyQt6 desktop application for comparing detection results from
three AI models on one high-resolution 4K image.

This project is intentionally separate from the main Musa AI application. It
does not use GeoTIFF processing, GIS rendering, raster layers, Leaflet, or map
libraries. It focuses only on visual comparison of already-generated detections
on a normal image.

## Features

- Loads one 4K image.
- Starts empty and waits for a real image/result file.
- Displays detection markers over the image.
- Preserves Musa AI-style detection record fields:
  - `id`
  - `image_name`
  - `class_name`
  - `latitude`
  - `longitude`
  - `confidence`
  - `pixel_x`
  - `pixel_y`
  - `health`
  - `source`
  - `duplicate_count`
  - `related_leaf_id`
  - `layer_keys`
  - extra metadata
- Compares two AI result groups:
  - Gemini Vision
  - Musa AI Trained YOLO Pipeline
- Configure and locally store a Gemini API key.
- Configure and locally store the two Musa AI model paths:
  - YOLOv8-seg leaf model for `full_leaf` and `cut_leaf`
  - YOLOv8 disease model for `black_sigatoka` and `panama`
- Toggle model visibility.
- Switch active model.
- Inspect marker metadata by clicking a point.
- Zoom, pan, reset view, fit image, and use keyboard shortcuts.

## Run

```powershell
cd model_comparison_desktop
pip install -r requirements.txt
python main.py
```

If `python` is not available on PATH, use your configured Python launcher or
virtual environment Python executable.

## Controls

- Mouse wheel: zoom in/out
- Left-drag: pan image
- Click marker: inspect detection metadata
- `Ctrl+O`: open another image
- `Ctrl+0`: fit image to view
- `Ctrl+R`: reset zoom

## Expanding Later

Use the toolbar to open a real 4K image. Detection markers appear once real
results are loaded or generated by future integration code. The renderer uses
`pixel_x` and `pixel_y` for placement, while keeping latitude and longitude
available for metadata comparison.

The built-in detection pipeline slices the image into overlapping `512x512`
tiles with `128px` overlap, similar to SAHI-style tiled inference. Because this
is a normal image and not a GeoTIFF, detections generated by the app keep
`latitude` and `longitude` fields but set them to `0.0` and mark metadata as
`"georeferenced": false`.

## Running Detection

1. Save a Gemini API key.
2. Select both trained model paths.
3. Open a real high-resolution image.
4. Choose one mapping mode in the toolbar:
   - `Gemini Vision`
   - `Musa AI YOLO Pipeline`
5. Click `Run Detection`.

The run processes:

- only the selected mapping mode
- elapsed time and class counts are shown after the run

The trained pipeline uses both local models as one detector:

```text
512x512 overlapping tile
  |
  |-- YOLOv8-seg leaf model -> full_leaf / cut_leaf
  |
  |-- YOLOv8 disease model  -> black_sigatoka / panama
  |
  -> combined trained_pipeline result set
```

## Result JSON Shape

`Load Results` expects a JSON file with model result groups:

```json
{
  "image": {
    "name": "your_4k_image.png",
    "width": 3840,
    "height": 2160
  },
  "models": [
    {
      "id": "gemini",
      "name": "Gemini Vision",
      "version": "gemini",
      "color": "#38bdf8",
      "description": "Gemini API results",
      "records": []
    },
    {
      "id": "trained_pipeline",
      "name": "Musa AI Trained YOLO Pipeline",
      "version": "leaf-model.pt + disease-model.pt",
      "color": "#22c55e",
      "description": "Combined two-stage local results",
      "records": []
    }
  ]
}
```

Each record should follow the Musa AI detection structure and include at least
`id`, `image_name`, `class_name`, `latitude`, `longitude`, `confidence`,
`pixel_x`, and `pixel_y`.
